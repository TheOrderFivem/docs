# Modules

Modules are bridged APIs that wrap framework-specific resources behind a single interface. Community Bridge auto-detects which resource you have installed and loads the correct implementation.

## Available Modules

|--------|------|-------------|
| [Framework](/community-bridge/modules/framework/compatibility) | Client & Server | Player data, jobs, metadata, account balances |
| [Inventory](/community-bridge/modules/inventory/compatibility) | Client & Server | Items, stashes, shops, metadata |
| [Banking](/community-bridge/modules/banking/compatibility) | Server | Account money management |
| [Boss Menu](/community-bridge/modules/bossmenu/compatibility) | Client & Server | Job management menus |
| [Clothing](/community-bridge/modules/clothing/compatibility) | Client & Server | Appearance menus and data |

## Usage Pattern

```lua
local Bridge = exports['community_bridge']:Bridge()

-- Access any module
Bridge.Framework.GetPlayerName()
Bridge.Inventory.AddItem(source, 'bread', 1)
Bridge.Notify.SendNotification(source, 'Title', 'Message', 'success', 5000)

-- Or get a single module
local Inventory = exports['community_bridge']:Inventory()
Inventory.AddItem(source, 'bread', 1)
```
